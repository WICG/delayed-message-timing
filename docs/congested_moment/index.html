Congested Moment

A congested moment occurs when the browser's main thread is overwhelmed by competing work—tasks, microtasks, and input events pile up faster than they can be processed, causing noticeable delays in responsiveness. These demos illustrate common scenarios that lead to congestion and help visualize its effects.

Demos

  • Concurrent Task Sources

    Shows how input events (mousemove) and timer tasks compete for the main thread. Move your mouse rapidly inside the box to observe scheduling delays when both task sources are active simultaneously.

  • Microtask Checkpoint

    Demonstrates how a flood of chained Promise microtasks can starve other queued tasks. After clicking Start, a postMessage task is enqueued but cannot run until the microtask checkpoint finishes—even though the click handler has already returned.
